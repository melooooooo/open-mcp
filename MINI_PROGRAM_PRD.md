# 银行帮小程序迁移需求文档

版本：v1.0  
日期：2026-05-26  
范围：将当前 Web 站点中面向 C 端用户的核心功能迁移到微信小程序，管理后台继续保留在 Web 端。

## 1. 背景与问题

当前 Web 站点已经从早期 OpenMCP 应用目录扩展为「银行招聘 求职指南」，首页聚合求职导航、经验分享、内推信息和最新招聘动态。用户在移动端访问招聘信息、收藏岗位、阅读面经和查看内推机会的频率较高，但 Web 端在微信生态内的登录、分享、留存和再次触达能力有限。

小程序需要解决 3 个核心问题：

1. 用户在微信内查看职位、面经、内推信息时路径过长，打开到完成收藏/阅读的步骤需要压缩到 3 步以内。
2. 当前 Web 端用户资产分散在「收藏职位、收藏内推、点赞经验、个人资料」中，小程序需要提供统一的移动端个人中心。
3. 当前内容聚合依赖搜索和筛选，小屏场景需要重构为更适合触摸操作的首页、列表、筛选抽屉和详情页。

## 2. 产品目标

### 2.1 首版目标

1. 迁移当前 Web 首页核心内容：求职导航、面经攻略、内推信息、招聘动态。
2. 支持微信小程序用户登录，并打通现有用户、收藏、点赞数据。
3. 支持职位、面经、内推的搜索、筛选、详情、收藏/点赞、分享。
4. 保留 Web 管理后台作为内容和审核入口，小程序只做用户端浏览与互动。

### 2.2 成功指标

上线后 30 天内达到以下指标：

| 指标 | 目标 |
| --- | --- |
| 小程序首页冷启动首屏可交互时间 | P95 <= 2.5s，弱网 4G P95 <= 4s |
| 搜索结果响应时间 | P95 <= 1.5s |
| 职位/内推/面经详情打开成功率 | >= 99.5% |
| 登录后收藏或点赞成功率 | >= 99% |
| 用户从首页进入任意详情页步骤 | <= 2 次点击 |
| 首月收藏/点赞用户占登录用户比例 | >= 25% |
| 首月小程序分享带来的详情页访问占比 | >= 10% |

| `/search` | 综合搜索职位、面经、求职导航 | P0 |

### 3.2 数据与接口基础

当前 Web 使用 Next.js、tRPC、Supabase、Drizzle，主要数据来源如下：

| 领域 | 数据表/接口线索 | 小程序复用方式 |
| --- | --- | --- |
| 招聘职位 | `job_listings`、`getJobListings`、`getLatestJobListings` | 新增小程序 BFF API 或复用服务端查询 |
| 求职导航 | `cp_job_sites`、`getJobs`、`getJobById` | 复用现有 Supabase 查询 |
| 面经攻略 | `finance_experiences`、`getExperiencesList`、`getExperienceBySlug` | 复用列表/详情能力，增加小程序字段裁剪 |
| 内推信息 | `scraped_jobs`、`getReferrals` | 复用列表/详情能力 |
| 收藏/点赞 | `userJobListingCollections`、`userCollections`、`userExperienceLikes` | 打通微信登录用户 ID 后复用 |
| 反馈 | `siteFeedbacks.create` | 小程序端新增反馈入口 |
| MCP 应用 | `mcpApps`、`mcpSearch`、`mcpRankings` | P2 迁移 |

## 4. 用户与场景

### 4.1 目标用户

| 用户 | 特征 | 核心任务 |
| --- | --- | --- |
| 应届生 | 关注银行、券商、保险、金融科技校招 | 查岗位、看截止时间、收藏岗位、读面经 |
| 社招候选人 | 关注社招、内推、企业招聘渠道 | 快速筛选职位、查看内推外链、保存机会 |
| 求职准备用户 | 暂无明确公司，依赖攻略和经验 | 搜索公司/岗位、阅读面经、点赞和分享 |
| 内容运营 | 使用 Web 后台维护数据 | 不使用小程序后台 |

### 4.2 核心用户故事

#### 首页发现

作为求职用户，我希望打开小程序后能立即看到最新职位、内推和经验，以便在 30 秒内找到值得查看的机会。

验收标准：

- Given 用户打开小程序首页，When 网络正常，Then 2.5 秒内展示搜索框和首屏内容。
- Given 首页有最新招聘数据，When 用户点击招聘动态卡片，Then 进入对应招聘详情页。
- Given 首页数据为空，When 用户访问首页，Then 显示空状态，不出现白屏或报错。

INVEST 检查：Independent、Negotiable、Valuable、Estimable、Small、Testable 均满足。

#### 职位搜索与筛选

作为求职用户，我希望按关键词、地点、企业性质和届次筛选职位，以便快速找到匹配我的岗位。

验收标准：

- Given 用户输入关键词并点击搜索，When 接口返回结果，Then 展示职位列表、总数和当前筛选条件。
- Given 用户选择多个企业性质，When 点击确认，Then 列表按所选性质过滤。
- Given 搜索无结果，When 列表为空，Then 显示「没有找到相关职位」和清空筛选按钮。
- Given 用户下拉到底部，When 还有下一页，Then 自动加载下一页，每页 20 条。

INVEST 检查：Independent、Negotiable、Valuable、Estimable、Small、Testable 均满足。

#### 职位收藏

作为登录用户，我希望收藏感兴趣的招聘或内推信息，以便之后在个人中心继续查看。

验收标准：

- Given 未登录用户点击收藏，When 系统检测无登录态，Then 唤起微信登录授权或登录引导。
- Given 登录用户点击收藏职位，When 请求成功，Then 收藏图标在 1 秒内变为已收藏状态。
- Given 已收藏职位再次点击，When 请求成功，Then 从收藏列表移除。
- Given 收藏接口失败，When 请求返回错误，Then 恢复原状态并显示错误提示。

INVEST 检查：Independent、Negotiable、Valuable、Estimable、Small、Testable 均满足。

#### 面经阅读与点赞

作为求职准备用户，我希望阅读面经详情并点赞有帮助的内容，以便表达反馈并沉淀我的阅读偏好。

验收标准：

- Given 用户打开面经详情，When 内容包含 Markdown 或 HTML，Then 小程序渲染标题、作者、公司、岗位、标签、正文和封面。
- Given 登录用户点击点赞，When 请求成功，Then 点赞数加 1 且按钮显示已点赞。
- Given 未登录用户点击点赞，When 用户完成登录，Then 返回当前面经详情并可继续点赞。
- Given 详情内容存在外链或图片，When 小程序渲染，Then 图片可预览，外链使用复制链接或内置 WebView 策略处理。

INVEST 检查：Independent、Negotiable、Valuable、Estimable、Small、Testable 均满足。

#### 个人中心

作为登录用户，我希望在个人中心查看资料、收藏和点赞内容，以便管理我的求职资产。

验收标准：

- Given 用户已登录，When 进入「我的」，Then 展示头像、昵称、城市、手机号或脱敏联系方式、加入时间。
- Given 用户有收藏，When 点击「我的收藏」，Then 展示内推收藏和招聘收藏，按收藏时间倒序排列。
- Given 用户有点赞面经，When 点击「我的点赞」，Then 展示点赞过的面经，按点赞时间倒序排列。
- Given 用户修改昵称/城市/性别，When 保存成功，Then 页面立即展示最新资料。

INVEST 检查：Independent、Negotiable、Valuable、Estimable、Small、Testable 均满足。

## 5. 功能需求

### 5.1 信息架构

小程序底部 Tab 建议 4 个：

1. 首页：搜索、推荐内容、招聘动态、内推、面经。
2. 职位：招聘职位列表和求职导航入口。
3. 经验：面经/攻略列表和详情。
4. 我的：资料、收藏、点赞、反馈。

二级页面：

- 综合搜索页
- 招聘详情页
- 求职导航列表/详情页
- 内推列表/详情页
- 面经详情页
- 反馈提交页
- 登录授权页

### 5.2 首页

功能要求：

1. 顶部展示品牌「银行帮」和搜索框，搜索占位文案为「搜索银行、岗位或关键词」。
2. 展示 8 个热门搜索词，默认沿用 Web：工商银行、建设银行、管培生、金融科技、数据分析、客户经理、风控、产品经理。
3. 展示求职导航模块，默认最多 8 条，排序规则沿用 Web：主流站点优先，其次按浏览量和创建时间。
4. 展示经验分享模块，默认最多 5 条，显示标题、作者、公司/岗位、浏览数。
5. 展示内推信息模块，默认最多 8 条，显示标题、发布时间、回复数、来源。
6. 展示最新招聘动态模块，默认最多 5 条，显示职位、公司、地点、行业、更新时间、新发布标记。
7. 首页所有卡片点击后进入对应详情页或外链处理页。

验收标准：

- 首页首屏接口数量不超过 3 个；如使用聚合接口，应一次返回首页所需的 4 个内容模块。
- 首页每个模块为空时显示独立空状态，不影响其他模块展示。
- 首屏搜索框、热门词、至少 1 个内容模块在 P95 2.5 秒内可见。

### 5.3 综合搜索

功能要求：

1. 支持搜索职位、面经、求职导航。
2. 搜索结果按分组展示：招聘职位、经验分享、求职导航。
3. 搜索关键词为空时展示搜索历史和热门搜索，不自动请求全部数据。
4. 搜索历史保存在本地，最多 10 条，用户可清空。
5. 支持从首页、职位页、经验页进入搜索并带入关键词。

验收标准：

- 输入关键词后 300ms 防抖发起搜索。
- 搜索结果分组总数为 0 时显示统一空结果。
- 点击任意结果进入详情页，返回后保留关键词和滚动位置。

### 5.4 招聘职位列表

功能要求：

1. 列表字段：职位名称、公司名称、工作地点、企业性质、行业、届次、学历要求、更新时间、截止时间。
2. 筛选字段：
   - 关键词：匹配职位名称和公司名称。
   - 工作地点：模糊匹配。
   - 企业性质：支持多选，枚举沿用 Web：央国企、民企、外企、事业单位、合资、社会组织、政府机关、国企、其他。
   - 届次：2024届、2025届、2026届、2027届、不限届。
3. 支持分页加载，每页 20 条。
4. 支持收藏/取消收藏。
5. 支持分享职位卡片到微信会话。

验收标准：

- 修改筛选后点击确认，列表从第一页重新加载。
- 列表总数展示与接口 count 一致。
- 下拉刷新后清空当前分页并重新加载第一页。
- 收藏状态与个人中心收藏列表一致。

### 5.5 招聘详情

功能要求：

1. 展示职位名称、公司名称、工作地点、企业性质、行业、届次、学历要求、专业要求、笔试要求、截止时间、申请方式、来源更新时间、备注。
2. 申请方式如果是 URL，提供「复制链接」和「打开外部链接」能力；如果是文本，提供复制。
3. 支持收藏、取消收藏、微信分享。
4. 支持相关职位推荐，推荐规则 P1 可按同公司、同企业性质、同地点匹配，最多 5 条。

验收标准：

- 不存在的职位 ID 显示缺省页，3 秒后返回职位列表。
- 截止日期已过时展示「已截止」标记。
- 点击申请方式复制后，系统剪贴板内容与展示文本一致。

### 5.6 求职导航

功能要求：

1. 展示招聘渠道站点，字段包括名称、Logo、描述、标签、浏览量、外部链接。
2. 支持快捷筛选：全部、今日更新、热门、校招、社招、实习。
3. 点击有 `website_url` 的站点时，使用复制链接或 WebView 打开；无外链时进入站点详情。
4. 记录站点浏览量。

验收标准：

- 筛选切换时不重新进入页面，列表在 300ms 内更新。
- 外部链接无法打开时提供复制链接按钮。

### 5.7 面经/攻略列表

功能要求：

1. 字段包括标题、作者、公司/组织、岗位、文章类型、行业、标签、阅读时间、浏览数、点赞数、发布时间、摘要、封面。
2. 支持行业筛选：全部行业、银行、券商、保险、运营商、科技/研发、其他行业。
3. 支持标签/关键词筛选，关键词匹配标签、标题、公司名、岗位。
4. 支持分页加载，每页 12 条。
5. 支持热门、置顶内容优先排序。

验收标准：

- 列表排序与 Web 一致：置顶、热门、发布时间倒序。
- 筛选条件展示为可关闭标签。
- 无结果时提供「查看全部经验」操作。

### 5.8 面经/攻略详情

功能要求：

1. 展示标题、作者、公司/团队、岗位、地点、发布时间、阅读时间、浏览数、点赞数、标签、封面、正文。
2. 正文支持 Markdown 和清洗后的 HTML 渲染。
3. 支持图片预览。
4. 支持点赞/取消点赞。
5. 支持分享给好友和生成分享卡片。
6. 保留 Web 端编辑能力，不在小程序首版提供编辑入口。

验收标准：

- 详情打开时浏览量递增 1 次，同一用户 30 分钟内重复打开是否去重要由服务端配置决定。
- 点赞状态进入页面时正确展示。
- 正文渲染不执行脚本，不展示未经清洗的危险 HTML。

### 5.9 内推广场

功能要求：

1. 展示内推列表，字段包括标题、发布时间、回复数、来源、公司名。
2. 支持分页加载，每页 30 条。
3. 支持收藏/取消收藏。
4. 支持内推详情，展示原文内容、来源、发布时间、外部链接。
5. 支持复制外链和微信分享。

验收标准：

- 列表按发布时间倒序。
- 收藏状态进入列表时批量返回，避免逐条请求。
- 外链为空时隐藏打开按钮。

### 5.10 个人中心

功能要求：

1. 微信登录后展示头像、昵称、用户 ID 后 8 位、性别、城市、联系方式、加入时间。
2. 支持编辑昵称、性别、城市、联系方式。
3. 展示统计：收藏数量、点赞数量、浏览历史数量。
4. 我的收藏包含招聘职位和内推信息，按收藏时间倒序。
5. 我的点赞包含面经/攻略，按点赞时间倒序。
6. 浏览历史 P1 实现，记录最近 100 条详情访问，按访问时间倒序，用户可清空。
7. 支持退出登录。

验收标准：

- 未登录用户进入「我的」时展示登录引导。
- 删除收藏后，列表项在 1 秒内消失，返回职位/内推页状态同步。
- 用户资料保存失败时保留表单输入，不丢失已编辑内容。

### 5.11 反馈

功能要求：

1. 支持反馈类型：新功能、Bug 报告、改进建议、其他。
2. 表单字段：标题、联系邮箱、详细描述、附件。
3. 附件最大 5MB，支持图片预览。
4. 反馈提交后写入现有 `site_feedbacks`。

验收标准：

- 标题和详细描述必填。
- 附件上传失败时允许用户不带附件继续提交。
- 提交成功后清空表单并返回「我的」页。

### 5.12 微信分享

功能要求：

1. 招聘详情、面经详情、内推详情支持 `onShareAppMessage`。
2. 分享标题分别使用职位标题、面经标题、内推标题。
3. 分享路径携带内容 ID/slug。
4. 面经详情 P1 支持生成分享海报，包含标题、公司/岗位、二维码。

验收标准：

- 分享卡片打开后能直接进入对应详情页。
- 未登录用户通过分享进入详情页可正常阅读公开内容。

### 5.13 P2 功能：MCP/AI 应用目录

如果首版之后继续迁移 OpenMCP 相关能力，范围如下：

1. 应用分类：MCP 客户端、MCP 服务器、AI 应用。
2. 应用列表：按类型、标签、子分类筛选。
3. 应用详情：简介、官网、GitHub、版本、许可证、统计数据、README 中文/原文、相关应用、推荐应用。
4. 标签页：展示标签下的应用数量和列表。
5. 排行榜：今日、昨日、上周、上月。
6. 应用提交、认领、建议修改保留 Web 或以 WebView 承载，不作为小程序原生首版。

## 6. 非功能需求

### 6.1 性能

| 场景 | 指标 |
| --- | --- |
| 首页首屏可交互 | Wi-Fi P95 <= 2.5s，4G P95 <= 4s |
| 列表接口响应 | P95 <= 1.5s |
| 详情接口响应 | P95 <= 1.2s |
| 图片加载 | 首屏图片体积 <= 200KB，列表缩略图 <= 80KB |
| 包体积 | 主包 <= 2MB，分包后总包 <= 微信平台限制 |

### 6.2 稳定性

1. 小程序核心 API 可用性 >= 99.5%。
2. 首页聚合接口失败时，允许单模块降级，不允许整页白屏。
3. 收藏、点赞、反馈等写操作必须返回可识别错误码。
4. 每个列表页必须具备加载、空、错误、重试状态。

### 6.3 安全与合规

1. 登录使用微信小程序登录码换取服务端会话，禁止在小程序端保存长期敏感密钥。
2. 个人联系方式展示需脱敏，编辑时才展示完整值。
3. Markdown/HTML 内容必须在服务端清洗，禁止小程序端直接渲染未清洗 HTML。
4. 文件上传限制类型和大小，附件最大 5MB。
5. 接口按用户身份校验收藏、点赞、资料修改权限。
6. 小程序隐私协议需声明收集头像、昵称、手机号、城市、收藏、点赞、浏览历史的用途。

### 6.4 兼容性

1. 支持微信 8.0 及以上版本。
2. 支持 iOS 最近 3 个大版本和 Android 主流机型。
3. 适配 320px 到 430px 宽度屏幕。
4. 长标题最多展示 2 行，详情页标题不截断。

## 7. 技术方案建议

### 7.1 架构

建议新增小程序 BFF 层，不直接让小程序调用 tRPC：

```text
微信小程序 -> 小程序 BFF REST API -> 现有业务服务/数据库
                         -> Supabase / Drizzle / tRPC 内部能力
```

原因：

1. 小程序对 HTTP、鉴权、错误码、上传和缓存有独立约束。
2. tRPC 客户端协议不适合作为小程序公开 API 契约。
3. BFF 可裁剪字段，降低移动端流量。

### 7.2 API 草案

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/mp/home` | 首页聚合数据 |
| GET | `/api/mp/search?q=` | 综合搜索 |
| GET | `/api/mp/job-listings` | 招聘职位列表 |
| GET | `/api/mp/job-listings/{id}` | 招聘详情 |
| POST | `/api/mp/job-listings/{id}/collection` | 收藏/取消收藏职位 |
| GET | `/api/mp/job-sites` | 求职导航列表 |
| GET | `/api/mp/job-sites/{id}` | 求职导航详情 |
| GET | `/api/mp/experiences` | 面经列表 |
| GET | `/api/mp/experiences/{slug}` | 面经详情 |
| POST | `/api/mp/experiences/{id}/like` | 点赞/取消点赞 |
| GET | `/api/mp/referrals` | 内推列表 |
| GET | `/api/mp/referrals/{id}` | 内推详情 |
| POST | `/api/mp/referrals/{id}/collection` | 收藏/取消收藏内推 |
| GET | `/api/mp/me` | 当前用户信息 |

### 7.3 统一响应格式

```json
{
  "code": "OK",
  "message": "success",
  "data": {},
  "requestId": "req_xxx"
}
```

错误示例：

```json
{
  "code": "UNAUTHORIZED",
  "message": "请先登录",
  "requestId": "req_xxx"
}
```

### 7.4 缓存策略

| 数据 | 缓存 |
| --- | --- |
| 首页聚合 | 服务端 60 秒，本地 5 分钟兜底 |
| 职位列表 | 服务端 60 秒，带筛选参数缓存 |
| 面经列表 | 服务端 5 分钟 |
| 面经详情 | 服务端 10 分钟，点赞状态单独合并 |
| 用户收藏/点赞状态 | 不缓存或本地乐观更新后服务端校准 |
| 热门搜索词 | 服务端 1 小时 |

## 8. 页面与交互规范

### 8.1 全局

1. 页面顶部使用小程序原生导航栏，标题分别为：首页、职位、经验、我的。
2. 列表页统一支持下拉刷新和触底加载。
3. 写操作使用乐观更新，但接口失败必须回滚。
4. 外部链接统一提供复制链接，允许时使用 WebView 打开。
5. 所有空状态必须包含下一步动作，例如清空筛选、返回首页、查看全部。

### 8.2 状态设计

每个页面至少覆盖 5 类状态：

1. 首次加载。
2. 加载成功。
3. 空数据。
4. 请求失败。
5. 登录失效。

## 10. 验收清单

### 10.1 功能验收

1. 未登录用户可浏览首页、搜索、列表和详情。
2. 登录用户可收藏招聘职位和内推信息。
3. 登录用户可点赞面经。
4. 个人中心展示收藏与点赞数据，数据与 Web 端一致。
5. 分享链接打开后进入正确详情页。
6. 所有列表页支持下拉刷新和触底加载。
7. 所有接口失败时有可见错误提示和重试入口。

### 10.2 数据验收

1. 招聘列表数量与 `job_listings` 查询结果一致。
2. 面经列表排序与 Web 一致。
3. 内推列表按 `publish_date` 倒序。
4. 收藏和点赞写入现有表，不新增重复关系表。
5. 用户资料更新后 Web 端和小程序端读取一致。

### 10.3 性能验收

1. 首页首屏 P95 <= 2.5s。
2. 搜索 P95 <= 1.5s。
3. 列表触底加载 P95 <= 1.5s。
4. 崩溃率 <= 0.3%。
5. JS 错误率 <= 0.5%。

## 11. 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| 现有 Web 直接调用 Supabase/tRPC，不适合小程序 | 小程序接口契约不稳定 | 增加 `/api/mp/*` BFF |
| 微信外链能力受限 | 申请方式、招聘站点无法直接打开 | 提供复制链接和 WebView 白名单策略 |
| HTML/Markdown 内容在小程序渲染差异 | 面经详情展示异常 | 服务端统一输出安全的 Markdown/节点 JSON |
| 用户体系迁移复杂 | 收藏/点赞无法关联 | 微信 openid/unionid 绑定现有 user 表 |
| 首页模块数据源多 | 首屏慢或局部失败 | 聚合接口并支持模块级降级 |
| 企业库仍是 mock 数据 | 小程序上线后可信度不足 | P2 前不作为核心入口 |

## 12. 未决问题

1. 微信登录是否必须绑定手机号，还是仅使用微信头像昵称即可。
2. 外部招聘链接是否申请 WebView 业务域名，还是首版只提供复制链接。
3. 面经正文最终以 Markdown、HTML 富文本还是结构化节点 JSON 下发。
4. 浏览历史是否落库，还是首版只保存在小程序本地。
5. MCP/AI 应用目录是否仍属于银行帮小程序的一部分，还是拆为独立入口。
6. 是否需要小程序消息订阅，用于职位更新和收藏岗位截止提醒。

## 13. 不在首版范围

1. Web 管理后台迁移。
2. 广告购买、在线支付和发票。
3. MCP 应用提交、认领、建议修改的原生小程序表单。
4. 面经富文本编辑器。
5. 企业对比、校友口碑、岗位提醒等当前 Web 标记为即将上线的功能。
6. 社交关注、粉丝、私信。
